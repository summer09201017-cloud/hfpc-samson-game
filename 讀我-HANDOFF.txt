這是什麼
--------
單一動作關的聖經小遊戲:參孫打獅子(士師記 14:5-6)。
純 vanilla ES modules + Canvas,零執行期相依、Web Audio 合成音效、可離線(PWA)。
架構沿用「約拿闖關」引擎。俯視角競技場 boss 戰:走位閃避 + 趁破綻反擊,6 下撕裂收尾。

★ 神學重點:聖經不說參孫多強壯,而說「耶和華的靈大大感動他」——力量出於神。
  不要把它改成「比力氣 / 練等變強」;不要加倚靠值能量條(使用者已明確決定不做)。

怎麼跑 / 測 / 建置(在這個資料夾)
---------------------------------
  npm install            第一次(裝 vite + sharp)
  npm run dev            開發,預設 http://localhost:5273/
  npm test               煙霧測試:文案 + 語法 + 狀態機/嵌入契約
  npm run test:offline   再加 build → PWA 離線就緒檢查
  npm run build          產生 site/(自寫 bundle,避開 Node 24 的 vite build 地雷)

本機地雷(務必遵守,詳見 CLAUDE.md):
  - 不要用 vite build(這台 Node 24 遞迴 cpSync 會無聲被殺);build 走 scripts/bundle-static.mjs。
  - .bat 一律純 ASCII + CRLF。
  - localhost 會自動 unregister Service Worker(改了才看得到);正式環境才註冊。
  - 新增 src/ 模組要加進 public/sw.js 的 CORE 清單(test:offline 會檢查)。


檔案在哪(一檔一責)
--------------------
  src/game.js       主迴圈 + 狀態機 + 戰鬥裁判 + 嵌入契約
  src/config.js   ★ 所有可調數值(調手感只動這裡:PHASES / maxHp / maxHearts / aimLockLead)
  src/scripture.js★ 經文與文案(和合本;改文案只動這裡)
  src/samson.js     參孫走位 + 反擊判定窗
  src/lion.js       獅子 AI 狀態機(含 _aimAt 紅線追蹤)
  src/renderer.js   所有 Canvas 繪製(只讀狀態)
  src/input.js      原始輸入(移動向量 + 攻擊邊緣 + 觸控)
  src/ui.js         DOM 覆蓋層(標題/勝利/失敗/暫停/彩蛋)
  src/audio.js      Web Audio 即時合成音效 + 背景音樂
  CLAUDE.md         開發筆記(含「現況」「玩法核心」「本機地雷」)
  roadmap.md        ★ 下一步要做什麼(已完成 vs 待做,按 CP 值排序)
